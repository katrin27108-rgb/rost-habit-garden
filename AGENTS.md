# Инструкции для агентов

Этот файл действует на весь репозиторий.

## Обязательное начало работы

Перед любыми изменениями агент обязан:

1. Полностью прочитать `MEMORY.md`.
2. Прочитать `docs/ARCHITECTURE.md` и выбрать подсистему по карте «запрос → код».
3. Выполнить `git status -sb`, проверить текущую ветку и сохранить чужие изменения.
4. Прочитать ближайшие тесты и публичные типы затрагиваемой подсистемы.
5. Зафиксировать задачу по шаблону `.agents/task-template.md` хотя бы в рабочем плане агента.

Нельзя начинать с переписывания сцены, хранилища или API без проверки их активной реализации.

## Источники истины

- Долговременный контекст и решения: `MEMORY.md`.
- Границы подсистем и потоки данных: `docs/ARCHITECTURE.md`.
- Запуск и команды: `README.md`.
- Production и секреты: `docs/DEPLOYMENT.md`.
- Схема данных: `db/schema.ts` и `drizzle/`.

Если документация расходится с выполняемым кодом, сначала подтвердите поведение тестом или трассировкой, затем исправьте документацию в том же Pull Request.

## Правила изменения кода

- Активный 3D-сад находится в `app/Garden3DPrototype.tsx` и `app/garden3d/`.
- `app/GardenPrototype.tsx` и `app/garden/` — старый 2D-прототип. Не развивайте его без прямого запроса.
- Не меняйте формат сохранённых привычек, API или D1 без миграции и обратной совместимости.
- Одна привычка соответствует одному растению; прогресс вычисляется доменной логикой, а не визуальной сценой.
- Локальная работа должна оставаться доступной без авторизации и сети.
- Не коммитьте `.env`, токены, D1 ID, временные пути, кеши или сгенерированный `dist/`.
- Не используйте `force push` для `main`.

## Завершение задачи

1. Запустите `npm run check` или документируйте конкретную недоступную проверку.
2. Пройдите `.agents/review-checklist.md`.
3. Обновите `MEMORY.md` только если появилось устойчивое архитектурное или продуктовое решение.
4. Обновите графы, если изменились границы, маршруты или поток данных.
5. В итоговом сообщении перечислите результат, проверки и оставшиеся риски.

`MEMORY.md` не является журналом сессий. Не записывайте туда переписку, временные статусы, догадки и секреты.
