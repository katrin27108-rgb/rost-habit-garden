# Инструкции для агентов

Этот файл действует на весь репозиторий.

## Перед тем как внести изменение на сайт

1. Прочитайте `MEMORY.md`: это краткая память проекта.
2. Откройте `docs/ARCHITECTURE.md` и найдите в таблице, где находится нужная часть сайта.
3. Выполните `git status -sb`. Не трогайте уже сделанные кем-то изменения.
4. Откройте рядом лежащий тест или тип, если он есть.

Если непонятно, в каком файле работать, сначала проверьте карту в `docs/ARCHITECTURE.md`, а не переписывайте сад, хранилище или API наугад.

## Источники истины

- Долговременный контекст и решения: `MEMORY.md`.
- Границы подсистем и потоки данных: `docs/ARCHITECTURE.md`.
- Запуск и команды: `README.md`.
- Production и секреты: `docs/DEPLOYMENT.md`.
- Схема данных: `db/schema.ts` и `drizzle/`.

Если документация расходится с кодом, сначала проверьте настоящее поведение тестом или в браузере, затем исправьте документацию вместе с изменением.

## Что важно не сломать

- Активный 3D-сад находится в `app/Garden3DPrototype.tsx` и `app/garden3d/`.
- `app/GardenPrototype.tsx` и `app/garden/` — старый 2D-прототип. Не развивайте его без прямого запроса.
- Не меняйте формат сохранённых привычек, API или D1 без миграции и обратной совместимости.
- Одна привычка соответствует одному растению; прогресс вычисляется доменной логикой, а не визуальной сценой.
- Локальная работа должна оставаться доступной без авторизации и сети.
- Не коммитьте `.env`, токены, D1 ID, временные пути, кеши или сгенерированный `dist/`.
- Не используйте `force push` для `main`.
- Работайте напрямую с `main`: внесите изменение, проверьте его, сделайте обычный коммит и отправьте его в `origin/main`. Pull Request не создавайте, если пользователь явно не попросил обратное.

## Когда изменение готово

1. Запустите `npm run check`. Если команда недоступна, честно укажите почему.
2. Быстро проверьте себя по `.agents/review-checklist.md`.
3. Если вы приняли правило, которое будет важно и в следующих задачах, обновите `MEMORY.md`.
4. Если менялись маршруты, части приложения или поток данных — обновите `docs/ARCHITECTURE.md`.
5. Сделайте коммит и отправьте его напрямую в `main`.
6. В конце коротко скажите, что изменилось и что проверено.

`MEMORY.md` не является журналом сессий. Не записывайте туда переписку, временные статусы, догадки и секреты.
