# Память проекта «Рост»

Последнее обновление: 2026-07-19.

## Продуктовая суть

«Рост» — игровой трекер привычек для компьютера и телефона. Каждая привычка создаёт ровно одно выбранное растение. Регулярные выполнения обеспечивают плавный рост на протяжении срока привычки; пропуски отражаются на здоровье растения и атмосфере сада, но не уничтожают пользовательский прогресс.

Основные ценности: тёплая поддержка без давления, понятная статистика, живой сад, автономная работа и синхронизация между устройствами.

## Текущее техническое состояние

- Стек: Node.js 22, vinext, React 19, TypeScript, Cloudflare Workers и D1.
- Production: `https://rost-habit-garden.katrin27108-rost.workers.dev`.
- Главный интерфейс: `app/page.tsx` и компоненты верхнего уровня в `app/`.
- Активная игровая сцена: `app/Garden3DPrototype.tsx` + `app/garden3d/` на Babylon.js, загружаемом браузером.
- `app/LivingGarden.tsx` — встроенный обзор на dashboard, а не замена полноэкранной сцене.
- `app/GardenPrototype.tsx` + `app/garden/` — устаревший 2D-прототип.
- Доменная логика: `lib/domain.ts`; сохранённая модель v2 и миграция: `lib/app-model.ts`.
- Offline-first очередь: `lib/offline-store.ts`.
- Активные серверные маршруты: `app/api/auth`, `app/api/garden`, `app/api/community`.
- Активная база: Cloudflare D1, binding `DB`; схема в `db/schema.ts`.
- Каталог `supabase/` сохранён как прежний вариант и не является production backend.

## Устойчивые решения

- Канонический GitHub-репозиторий: `katrin27108-rgb/rost-habit-garden`.
- GitHub Pages — архивная статическая версия, не production.
- Production-деплой идёт в Cloudflare Workers только после успешной CI-проверки `main`.
- Локальные привычки работают без аккаунта; аккаунт добавляет межустройственную синхронизацию и доступ к разрешённым садам.
- Рост и здоровье растения рассчитываются из дат, расписания и выполнений. Визуальная сцена только отображает результат.
- Размещение растения хранится вместе с привычкой и должно переживать перезапуск и синхронизацию.
- Секреты и реальные инфраструктурные ID хранятся вне Git.

## Известные ограничения и технический долг

- Активная сцена использует собственный слой загрузки Babylon.js и бесплатные ассеты; качество и лицензии новых ассетов необходимо проверять отдельно.
- В репозитории остаётся старый 2D-код, поэтому агент должен явно выбирать активную реализацию.
- Визуальные React/Babylon-модули имеют зафиксированный lint-долг: соответствующие правила временно выводят предупреждения, а не блокируют CI. Новые изменения не должны увеличивать их количество.
- Долговечность офлайн-очереди и разрешение конфликтов между устройствами требуют отдельных интеграционных тестов.
- Социальный просмотр сада реализован на уровне API, но будущие права редактирования должны проектироваться отдельно от права просмотра.
- `.openai/hosting.json` поддерживает прежнюю Sites-инфраструктуру разработки; канонический production остаётся Cloudflare Workers из GitHub Actions.

## Как обновлять этот файл

Добавляйте только подтверждённые решения, действующие дольше одной задачи. При изменении решения заменяйте старую запись, а не накапливайте противоречия. Никогда не добавляйте токены, пароли, D1 ID, персональные данные, локальные абсолютные пути или пересказ диалога.
